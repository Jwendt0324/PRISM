# Examples — Real-World PRISM Implementations

These files are from a real agency's PRISM deployment. They show what filled-in SOPs, client automations, and scripts look like in practice.

**These are NOT required.** They're here for reference only.

## What's Here

### Client-Specific SOPs
- `sops/parts-agent-operations.md` — Automated parts pricing agent for an appliance repair client
- `sops/field-service-parts-inquiry.md` — Manual parts inquiry process using a field service platform

### Client-Specific Scripts  
- `scripts/parts-agent/` — Python automation for parts pricing lookups
- `scripts/scheduling-agent/` — Route optimization for field service scheduling

## How to Use These

1. Read them to understand how a real SOP is structured
2. Create your own versions in `sops/client-work/` for your clients
3. Don't modify these files — create new ones based on the patterns you see
