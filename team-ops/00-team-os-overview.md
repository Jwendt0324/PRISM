## How Work Flows

Every client engagement follows the same pipeline. No exceptions.

### 1. Sales ([Your Name] / [Your Mentor/Advisor])
- Lead comes in through outreach, referral, or inbound
- [Your Name] or [Your Mentor/Advisor] runs the sales conversation
- Deal closes on KP Sprint ($6k) or AI Apprentice ($2.5k)

### 2. Intake ([Your Ops Admin])
- [Your Ops Admin] creates a Basecamp project from the Sprint template
- RACI is assigned for every deliverable
- Client onboarding questionnaire is sent
- 30-day clock starts when onboarding materials are received

### 3. Production (Team)
- Team executes against the Sprint checklist
- Content Factory assets are produced
- Daily progress tracked in Basecamp
- EOD reports submitted by every team member working on the project

### 4. QA ([Your Name])
- All deliverables reviewed against quality standards before client delivery
- Influence Report Card generated (before/after comparison)
- No deliverable goes out without QA sign-off

### 5. Delivery
- Final assets delivered to client
- Walkthrough call scheduled if needed
- Client feedback captured

### 6. Case Study
- Every completed Sprint produces a case study
- Before/after metrics documented
- Case study feeds sales pipeline

## Communication Norms

### Tools

| Tool | Used For |
|------|----------|
| **Basecamp** | All project work. Task assignments, file sharing, project discussions. This is the system of record. |
| **Email** | Direct communication with clients and external parties. Internal communication only when it doesn't belong in Basecamp. |

### Cadence

| Ritual | When | Who | What |
|--------|------|-----|------|
| **EOD Report** | Daily, end of day | Everyone actively working | What you did today, what's blocked, what's next. Posted in Basecamp. |
| **MAA Report** | Weekly, Fridays | Everyone | Metrics, Activities, Actions — what moved this week, what moves next week. |
| **Weekly Standup** | Tuesdays, 30 minutes | Core team | Status updates, blockers, priorities for the week. No rambling. |

### Communication Rules

- If it's about a project, it goes in Basecamp. Not Slack. Not text. Basecamp.
- If you're blocked, say so immediately. Don't wait for a standup.
- EOD reports are not optional. They take 5 minutes. Do them.
- When in doubt, over-communicate. Silence is not a status update.

## Core Values

These aren't motivational posters. They're operating rules.

### LDT — Learn, Do, Teach

Everything we do follows this cycle. You learn a skill. You execute it on real work. Then you teach it to someone else (or document it so someone else can learn from what you did). If you only learn, you're a student. If you only do, you're a cog. If you only teach, you're a theorist. LDT means you do all three, in that order, always.

### CCS — Content, Checklist, Software

Every process at [Your Agency] lives in one of three forms:
- **Content** — An article, video, or training that explains the concept
- **Checklist** — A step-by-step list that ensures consistent execution
- **Software** — A tool or automation that removes manual steps

If a process doesn't exist in at least one of these forms, it doesn't exist at all. Our goal is to move every process from Content → Checklist → Software over time.

### Compound, Don't Campaign

We don't run campaigns. Campaigns end. We build systems that compound. Every piece of content, every Sprint delivered, every apprentice trained adds to the machine. If what you're doing doesn't compound, question whether you should be doing it.

### Every Sprint Yields a Case Study

No Sprint is complete until the case study is written. This is non-negotiable. The case study is not a nice-to-have — it's the fuel for the next sale.

### No Custom Work Inside a Sprint

The Sprint is a fixed playbook. If a client wants something outside the playbook, it's a separate engagement. Custom work inside a Sprint breaks the system, blows up timelines, and destroys margins. The answer is always: "We can absolutely do that — as a separate project after the Sprint."
