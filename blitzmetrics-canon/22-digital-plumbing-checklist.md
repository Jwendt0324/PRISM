# Digital Plumbing Checklist — [Methodology Partner] Canon

**Source:** "Digital Plumbing" definitive article ([Your Mentor/Advisor], March 2026), Master Operations Guide (v12.5), Master Access Checklist (v12.18), AI Business Audit (Luke Crowson), Are You Googleable ([Vertical Lighthouse])
**Version:** 1.0 | **Extracted:** 2026-04-01

---

## Core Principle

"Trying to fill a bathtub with the drain open."

That is what running ads, creating content, or investing in SEO looks like without Digital Plumbing in place. Digital Plumbing is the foundation that everything else builds on. It is Stage 0 of the Content Factory and Phase 1 of the Social Amplification Engine. No money should be spent on marketing until plumbing is verified.

Digital Plumbing answers one question: **Can we measure what matters?** If tracking is broken, optimization is impossible. If optimization is impossible, money is wasted.

## The 5 Pillars

### Pillar 1: Google Business Profile

The GBP is the single most important digital asset for any local business.

#### Foundation
- [ ] GBP claimed and verified with correct business owner (not a vendor)
- [ ] Business name matches legal name exactly (no keyword stuffing)
- [ ] Correct primary category selected
- [ ] Correct secondary categories (up to 9)
- [ ] Physical address verified (not virtual office)
- [ ] Phone number active, routing correctly, and tracked
- [ ] Business hours accurate and maintained
- [ ] Website URL pointing to correct landing page

#### Content & Media
- [ ] Profile photo is real (owner or branded image, not stock)
- [ ] Cover photo shows real work, team, or facility
- [ ] 50+ photos minimum (real job sites, team, vehicles, office)
- [ ] Photos geotagged to service area locations
- [ ] Video content uploaded directly to GBP
- [ ] Regular GBP posts (minimum weekly)
- [ ] Posts include real job updates, not generic tips

#### Reviews
- [ ] 100+ reviews for map pack competitiveness
- [ ] 4.5+ star average
- [ ] Response to every review (positive and negative)
- [ ] Reviews mention specific services and locations
- [ ] Recent reviews within last 30 days

#### Technical
- [ ] NAP matches website exactly (character for character)
- [ ] Service area defined correctly
- [ ] All services listed with descriptions
- [ ] Booking/appointment URL configured
- [ ] Products section populated (if applicable)
- [ ] Q&A section seeded with common questions and answers

#### Security
- [ ] Only authorized users have access
- [ ] Previous vendor access revoked
- [ ] Call tracking number verified (not hijacked)
- [ ] Weekly monitoring for unauthorized suggested edits

### Pillar 2: Website Health

The website is the central hub connecting all digital properties.

#### Speed & Performance
- [ ] Page load time under 3 seconds on mobile
- [ ] Core Web Vitals passing (LCP, INP, CLS)
- [ ] Mobile-responsive design (test on actual devices, not just simulators)
- [ ] HTTPS certificate active and properly configured
- [ ] No mixed content warnings

#### Technical SEO
- [ ] Schema markup implemented:
  - LocalBusiness (for company)
  - Person (for owner/founder)
  - FAQ (on relevant pages)
  - AggregateRating (if reviews exist)
  - Service (for each service offered)
  - GeoCoordinates (for physical location)
- [ ] XML sitemap submitted to GSC and functional
- [ ] Robots.txt properly configured
- [ ] No orphan pages (every page linked from at least one other page)
- [ ] Canonical tags on all pages

#### Content Structure
- [ ] Homepage clearly states WHO, WHAT, WHERE
- [ ] Service pages for each core offering
- [ ] Location-specific pages for each service area (with unique content, not template swaps)
- [ ] Real photos and videos, not stock images
- [ ] Clear calls to action on every page

#### Contact Page Optimization

The contact page is often the highest-leverage page on a local business website. Joel Headley (former Google) confirmed Google crawls contact pages for entity verification.

- [ ] H1 includes business name, primary service, and geography
- [ ] Service area lists every city served by name with internal links
- [ ] Entity authority block consolidating credentials (name, license number, years in business, rating, certifications)
- [ ] FAQ section with 5-10 most common customer questions + FAQ schema
- [ ] Full schema markup (LocalBusiness, Organization, AggregateRating, Service, FAQ, GeoCoordinates)
- [ ] Map embed with city name text nearby
- [ ] Trust signals near the contact form (response time promise, review snippet, technician photo)

### Pillar 3: Analytics & Tracking

If you cannot measure it, you cannot improve it.

#### Google Analytics 4
- [ ] GA4 property created and tracking code installed
- [ ] Key events configured (form submissions, phone clicks, chat initiations)
- [ ] Conversion events marked and verified
- [ ] Enhanced measurement enabled (scroll, outbound clicks, site search)
- [ ] Connected to Google Ads for audience sharing

#### Google Search Console
- [ ] Property verified (DNS or HTML tag method preferred)
- [ ] Sitemap submitted
- [ ] Core Web Vitals reviewed and passing
- [ ] Manual actions checked (none present)
- [ ] International targeting configured (if applicable)

#### Google Tag Manager
- [ ] GTM container created and published to website
- [ ] All tags firing correctly (verify with Tag Assistant)
- [ ] Meta Pixel deployed through GTM
- [ ] Conversion linker tag active
- [ ] Custom event tags for key user actions

#### Meta (Facebook) Pixel
- [ ] Pixel installed and firing on all pages
- [ ] Standard events configured (PageView, Lead, Contact, Purchase as applicable)
- [ ] Verified in Facebook Events Manager
- [ ] Domain verified in Business Manager

#### Call Tracking
- [ ] Call tracking number active (CallRail, ServiceTitan, or equivalent)
- [ ] Dynamic number insertion on website
- [ ] Call source attribution working (organic, Maps, paid, direct)
- [ ] Call recording enabled (with legal compliance)
- [ ] Integration with CRM or dispatch system

---

### Pillar 4: Social Profiles

Every social profile is an entity signal. Consistency across profiles strengthens the entity.

#### Required Profiles (Company Brand)
- [ ] Facebook Business Page -- claimed, branded, active
- [ ] Instagram Business Account -- connected to Facebook
- [ ] YouTube Channel -- branded, with content uploaded
- [ ] LinkedIn Company Page -- complete with description

#### Required Profiles (Personal Brand / Owner)
- [ ] Facebook personal profile -- public posts enabled
- [ ] LinkedIn personal profile -- complete with XYZ headline format
- [ ] YouTube (if separate from company channel)

#### Extended Profiles (as relevant)
- [ ] TikTok Business Center + Page
- [ ] Twitter/X
- [ ] Crunchbase
- [ ] Pinterest
- [ ] Reddit
- [ ] Medium
- [ ] About.me

#### Consistency Checklist
- [ ] Same profile photo across all platforms
- [ ] Same bio/description (adjusted for platform character limits)
- [ ] Same website URL linked
- [ ] Same phone number and address
- [ ] All profiles link back to the website (bidirectional linking)

---

### Pillar 5: Domain & Email

Ownership and professional communication.

#### Domain
- [ ] Domain registered to the business owner (not a vendor or developer)
- [ ] Registrar access credentials documented and secured
- [ ] Auto-renewal enabled
- [ ] Domain NOT expiring within 12 months without renewal plan

#### Professional Email
- [ ] Business email on the company domain (not @gmail.com or @yahoo.com)
- [ ] Google Workspace or Microsoft 365 configured
- [ ] Email signatures include name, title, phone, website

#### Email Authentication
- [ ] SPF record configured in DNS
- [ ] DKIM signing enabled
- [ ] DMARC policy set (at minimum p=none for monitoring; p=quarantine or p=reject for enforcement)
- [ ] Email deliverability tested (no blacklists)

## Access Checklist Reference

The Access Checklist is the FIRST operational deliverable after client onboarding. It covers Company Brand AND Personal Brand columns for 30+ platforms.

### Required Access (Bold = Non-Negotiable)

| Category | Platforms |
|----------|-----------|
| **Google** | Google Ads, GTM, GBP, GA4, GSC |
| **Meta** | Business Manager, Facebook Profile, Page, Ad Account, Instagram, 2FA method |
| **Website** | WordPress/Squarespace/HubSpot admin, hosting panel |
| YouTube | Channel access via Brand Account |

| Twitter/X | Account access |
| Others | Reddit, Crunchbase, Pinterest, Snapchat, Twitch, Discord, Quora, Medium, Vimeo, Alignable, Flickr |

### Access Documentation Rules
- Every credential stored in a secure password manager (not a Google Sheet)
- Screenshot verification of access granted (not just "I sent the invite")
- Previous vendor access revoked before new work begins
- Access audit quarterly

## Digital Plumbing Verification Process

### Step 1: Audit Current State
Run through all 5 pillars and mark each item as passing, failing, or not applicable.

### Step 2: Prioritize Fixes
1. Anything that blocks measurement (analytics, tracking, pixels)
2. Anything that blocks discovery (GBP issues, schema, NAP inconsistency)
3. Anything that blocks performance (site speed, mobile, HTTPS)
4. Anything that blocks engagement (social profiles, email deliverability)

### Step 3: Execute Fixes
Follow the priority order. Do not move to content or ads until Plumbing is verified.

### Step 4: Verify
- Tag Assistant confirms all tags firing
- Facebook Events Manager shows pixel activity
- GA4 real-time shows active users
- GSC shows indexed pages
- Call tracking shows source attribution
- Site speed under 3 seconds

### Step 5: Document
Record all access credentials, tracking IDs, and verification screenshots. This becomes the client's Plumbing file.

## Human Requirements

1. **GBP verification requires physical mail or phone call** -- cannot be fully automated
2. **Domain ownership transfer requires human authorization** -- registrars require identity verification
3. **Access requests to client platforms require human communication** -- clients must grant access; bots cannot request it
4. **Call tracking setup requires coordination with phone provider** -- porting numbers or adding tracking numbers involves human support calls
5. **Schema validation requires human review** -- automated generators can produce syntactically valid but semantically incorrect markup
6. **Security audit of existing access** -- determining who should and should not have access requires human judgment about organizational relationships
