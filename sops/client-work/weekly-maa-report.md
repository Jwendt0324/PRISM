# Weekly MAA Report SOP

## Purpose
Deliver a consistent, numbers-driven weekly report that shows clients (and [Your Mentor/Advisor]) what happened, what it means, and what happens next — in under 5 minutes.

## When to Use
- Every Friday for every active client engagement
- When [Your Mentor/Advisor] or a client requests a status update (use the MAA format even ad hoc)
- After a major campaign change to document before/after metrics

## Process

1. **Pull current metrics.** Gather actual numbers for the reporting period:
   - Website traffic (sessions, users, top pages)
   - Search rankings (target keywords, position changes)
   - Content published (count, titles, platforms)
   - Engagement (social likes, comments, shares, video views)
   - Any campaign-specific KPIs (ad spend, conversions, CPL)

2. **Write the Metrics section.** Report **specific numbers**, not descriptions. Every metric needs a number and a comparison.
   - Bad: "Traffic increased this week."
   - Good: "Traffic increased 23% to 1,240 sessions (vs. 1,008 last week)."

3. **Write the Analysis section.** Explain **why** the numbers look the way they do. Connect cause to effect. Identify what is working, what is flat, and what is declining.
   - [Your Mentor/Advisor] expects analysis, not task lists. "We published 3 articles" is a task. "The 3 articles targeting long-tail hearing aid keywords drove 340 new organic sessions, up from 180 last week" is analysis.
   - Call out the weakest area explicitly. Do not bury bad news.

4. **Write the Action section.** List next steps with **owners** and **deadlines**.
   - Every action item must have a person's name and a date.
   - At least one action item should address the weakest metric from the Analysis section.
   - Format: `[Action] — [Owner] — [Deadline]`

5. **Post the MAA to the Basecamp Updates thread** (client-visible). The client and [Your Mentor/Advisor] should see it without having to ask.

6. **If metrics are not available or incomplete,** post the MAA anyway with what you have and note what is missing and when it will be updated. Never skip a week silently.

## MAA Report Template

```
## MAA Report — [Client Name] — Week of [Date]

### Metrics
- Website sessions: [number] ([+/- %] vs. last week)
- Top keyword rankings: [keyword]: [position] ([change])
- Content published: [count] ([list titles])
- Engagement: [platform]: [numbers]
- [Campaign KPI]: [number]

### Analysis
[2-4 sentences connecting metrics to actions taken. What is working and why. What is flat or declining and why. Identify the weakest area.]

### Action
- [Action item] — [Owner] — [Deadline]
- [Action item] — [Owner] — [Deadline]
- [Action item] — [Owner] — [Deadline]
```

## Quality Checks
- [ ] Metrics section contains actual numbers with week-over-week comparisons
- [ ] Analysis section has insights explaining the "why" behind the numbers (not just descriptions of tasks completed)
- [ ] Action section has at least 3 items, each with an owner and a deadline
- [ ] At least one action item targets the weakest metric area
- [ ] Posted to Basecamp Updates thread by Friday EOD

## Common Pitfalls
- **Treating the MAA as a task list.** "We wrote 3 articles and posted 5 social updates" is a status update, not analysis. [Your Mentor/Advisor] will push back. Always connect output to outcomes.
- **Missing a week without notice.** If you cannot submit Friday, post a note in the Updates thread saying when it will come. Never go silent — that is how you lose trust.
- **Vague metrics.** "Traffic increased" is useless. "Traffic increased 23% to 1,240 sessions" is useful. Always include the actual number and the comparison.
- **No action items tied to the weakest area.** If organic traffic is flat but you only list content actions, you are ignoring the problem. The Action section must address whatever is underperforming.
